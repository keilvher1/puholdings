# 포항연합기술지주 통합 시스템 운영 매뉴얼

> 최종 갱신: 2026-07-21 · 대상: 운영 담당자(관리자) 및 입주기업 안내용

---

## 1. 시스템 구성과 주소

| 영역 | 주소 | 대상 |
|---|---|---|
| 공개 홈페이지 | https://www.puholdings.co.kr | 누구나 |
| 관리자 CMS | https://www.puholdings.co.kr/admin | 운영 담당자 |
| 입주기업 포털 | https://www.puholdings.co.kr/portal | 입주기업 (계정 발급 필요) |

- 공개 홈페이지: 회사소개·조직·포트폴리오·최신소식·문의. 콘텐츠 대부분을 관리자 CMS에서 편집할 수 있습니다.
- 관리자 CMS: 사이트 콘텐츠 편집 + 입주기업/호실/관리비/프로그램/메일 운영.
- 입주기업 포털: 각 기업이 본인 청구서 확인·프로그램 신청·자료 제출.

## 2. 계정 안내

### 2-1. 관리자 계정

| 항목 | 내용 |
|---|---|
| 로그인 주소 | https://www.puholdings.co.kr/admin/login |
| 현재 계정 | `admin1@puholdings.com` (이름: 관리자) |
| 비밀번호 | 계정 개설 시 설정한 값 (시스템에는 암호화 저장되어 조회 불가) |
| 세션 유지 | 7일 |

- **관리자 추가**: 기존 관리자로 로그인한 상태에서 `/admin/setup` 페이지에서 추가할 수 있습니다. (보안상, 관리자가 이미 존재하면 비로그인 상태로는 계정을 만들 수 없습니다)
- **비밀번호 분실**: 화면에서 재설정하는 기능은 없습니다. 개발 담당자에게 요청해 DB에서 재설정해야 합니다.

### 2-2. 입주기업 포털 계정

| 항목 | 내용 |
|---|---|
| 로그인 주소 | https://www.puholdings.co.kr/portal/login |
| 계정 단위 | 기업당 1개 |
| 발급 방법 | 관리자 → **입주기업** 메뉴 → 해당 기업의 **[계정 발급]** 버튼 |
| 현재 상태 | **아직 발급된 계정 없음** (기업 27곳 등록 완료, 계정은 필요 시 발급) |

**발급 절차** (관리자):
1. `/admin/tenants` → 기업 행의 **[계정 발급]** 클릭
2. 로그인 이메일(기본: 담당자 이메일)·담당자 이름 입력 → 발급
3. **임시 비밀번호가 화면에 1회만 표시**됩니다 → [복사] 버튼으로 복사해 기업에 전달 (창을 닫으면 다시 볼 수 없고, [비밀번호 초기화]로 새로 발급해야 합니다)
4. 기업이 임시 비밀번호로 첫 로그인하면 **비밀번호 변경이 강제**됩니다

- 퇴거 처리된 기업은 즉시 포털 접근이 차단됩니다.
- 메일 발송 환경(7장)이 설정돼 있으면 발급 시 계정 안내 메일이 자동 발송됩니다.

## 3. 관리자 메뉴별 사용법

사이드바 순서대로 설명합니다.

### 3-1. 대시보드 (`/admin`)
소식·포트폴리오·통계·문의 수와 **호실 현황 요약**(전체/입주/공실/입주율)을 보여줍니다. 카드를 누르면 해당 메뉴로 이동합니다.

### 3-2. 사이트 콘텐츠 (`/admin/site`)
공개 홈페이지에 노출되는 내용을 편집합니다. 탭 구성:

| 탭 | 편집 내용 | 반영 위치 |
|---|---|---|
| 설정·텍스트 | 메인 히어로 문구, 연락처(주소·전화), 운영도 소개 문단 | 홈, 하단 푸터, 문의, 회사소개 |
| 연혁 | 연도·날짜·내용 (연도별 자동 그룹) | 회사소개 > 연혁 |
| 조직 구성원 | 팀(경영진/전략기획실/투자사업팀/창업보육팀/벤처파트너)별 이름·직위·역할·이력 | 조직 페이지 |
| 유관기관 | 기관명·로고(업로드 가능)·설명 | 조직 페이지 협력기관 표 |
| 운영도 · 활동 | 한동대/포항연합기술지주/외부 네트워크 활동 항목 | 회사소개 > 핵심 기능 및 역할 |

- 각 항목은 **표시 토글(눈 아이콘)**로 숨길 수 있고, **정렬 순서**를 지정할 수 있습니다.
- 저장 즉시 공개 페이지에 반영됩니다.

### 3-3. 최신 소식 / 팝업 / 포트폴리오 / 통계 / 문의
- **최신 소식**: 글 작성·수정, 파일 첨부(PDF/HWP/이미지 등, 파일당 20MB). 첨부는 홈페이지에서 다운로드·미리보기 가능.
- **팝업**: 기간·우선순위를 지정해 홈페이지 팝업 노출. 소식과 연결 가능.
- **포트폴리오**: 투자기업 정보·로고 관리.
- **통계**: 홈 화면 숫자(입주기업 수, 누적 매출 등) 수정.
- **문의**: 홈페이지 문의 접수 내역 열람. (메일 환경 설정 시 접수 알림 메일 자동 발송)

### 3-4. 입주기업 (`/admin/tenants`)
기업 정보(사업자번호·대표·호실·연락처·입주/퇴거일·상태) 관리와 **포털 계정 발급**(2-2 참고).
※ 관리비의 계약·호실과는 별개 화면입니다. 관리비 청구 대상은 3-6의 "계약"이 기준입니다.

### 3-5. 호실 현황 (`/admin/rooms`)
층별 호실 보드. 색으로 상태 구분:

| 색 | 상태 |
|---|---|
| 파랑 + 기업명 | 입주중 |
| 회색 | 공실 |
| 주황 + D-day | 퇴실 예정 (계약 종료일 기준) |
| 빗금 | 사용불가 (호실 상태를 maintenance로 지정한 경우) |

상단 요약(전체·입주·공실·입주율)과 필터 제공. 카드를 누르면 상세 정보가 열립니다. 입주 계약 등록·퇴실 처리는 **관리비 정산 > 설정 > 계약 관리**에서 합니다.

### 3-6. 관리비 정산 (`/admin/billing`) ⭐ 핵심 업무

기존 엑셀 정산표를 대체합니다. 탭 3개: **월 마감 / 청구서 / 설정**

#### 개념 정리
- **청구 시점 규칙**: 청구월 M의 청구서 = **M월 임대료 + (M−1)월 사용 전기료**. 전기료는 항상 한 달 늦게 청구됩니다. (예: 5월 청구서 = 5월 임대료 + 4월 전기료)
- **임대료** = 부과평형 × 평당단가 (갱신 21,000원 / 신규(비갱신) 20,000원 — 계약별 수정 가능)
- **관리비** = 호실당 정액 15,000원 (계약별 수정 가능)
- 공급가액 = 부가세포함액 ÷ 1.1 반올림, 부가세 = 차액 → **합계가 1원도 어긋나지 않습니다** (엑셀의 ±1 수동 보정 불필요)
- **전기료 배분**: 한전 총액 → ① 공장동 실사용(A, 검침) ② 사무실 면적별(B, 10평당 단가) ③ 센터부담(C) 3단 배분
- **공실은 자동 제외**됩니다 (청구 생성, 전기료 분모 모두)

#### 매월 마감 루틴 (월 마감 탭, 4단계 마법사)
1. **검침 입력** — 상단에서 전기 **사용월** 선택 → 계량기 4개(공장동 전체/F101/F103/냉난방기)의 당월 누적 지침 입력 → [검침 저장]. 전월 지침·사용량·청구 미리보기가 실시간 표시되고, 당월<전월이면 빨간 경고.
   - 냉난방기 사용량은 F101·F103에 절반씩 자동 가산, 청구액은 천원 미만 절사(엑셀과 동일).
2. **전기료 배분** — 한전 총 전기료·kWh 단가(기본 102원)·면적별 배분율(기본 0.70) 입력 → [계산·저장]. 배분 검산표(A/B/C)가 표시되면 **10평당 실계산 단가를 확인하고 청구단가를 확정**([제안값] 버튼 = 10원 단위 반올림).
3. **청구서 생성** — [청구서 생성] 클릭 → 기업별 초안(draft) 생성. 스킵된 기업(이미 발행됨 등)은 사유와 함께 표시. 초안은 **청구서 탭에서 행별 수정 가능**.
4. **발행** — [일괄 발행 + PDF 메일] → 상태가 issued로 바뀌고, **기업별 청구서 PDF가 생성**되어 세금계산서 메일(없으면 담당자 메일)로 발송됩니다. 발행 후에는 항목 수정 불가.

중간에 끊겨도 됩니다 — 입력값은 저장되며 다시 들어오면 이어서 진행됩니다.

#### 청구서 탭
- 월·상태(작성 중/발행됨/납부 완료/연체) 필터, 행 클릭 → 상세(항목 내역·PDF 미리보기)
- **납부 확인**: 발행된 청구서에서 이체일 입력 → [납부 확인] → paid
- **수동 청구서 만들기**: 정산표 밖 수기 청구(퇴거 정산 등). 기업·청구월·자유 항목 입력
- 납기 3일 전 자동 리마인더, 납기 경과 시 자동 연체 전환+안내 메일 (7장의 CRON 설정 필요)

#### 설정 탭
- **호실 관리**: 호실 추가·수정 (본관/공장동, 평형, 사용불가 지정)
- **계약 관리**: 기업×호실 계약 등록·수정.
  - 계약구분(갱신/비갱신) 선택 시 평당단가가 자동 제안됩니다
  - 전기 방식: 면적별(사무실) / 실사용(공장동 F101·F102·F103)
  - 기준 보증금 = 평당 20만원 × 실제평형 자동 계산, 실보증금은 수기 입력
  - **[퇴실] 버튼**: 종료일·마지막 달 청구 방식(전액/일할/없음)·보증금 반환액 입력 → 미납 상계 내역이 표시되고 호실은 자동 공실 전환
- **데이터**: 
  - 기존 정산 엑셀(xlsx) 업로드 → 기업·계약·호실 자동 추출 미리보기 → [반영] (기존 기업은 이름으로 매칭되어 중복 생성되지 않음)
  - 월별 정산표 엑셀 내려받기

### 3-7. 프로그램 (`/admin/programs`)
지원사업·교육 공고 → 신청 → 파일 제출 → 검토 흐름.

1. **공고 등록**: 제목·내용·신청기간·제출마감·공고문 첨부. 상태 **draft(작성 중)** 로 저장
2. **모집 시작**: 상태를 **open**으로 바꾸면 → **전체 입주기업에 공지 메일 자동 발송** (확인창이 뜸)
3. **신청 관리**: 공고 클릭 → "신청 현황" 탭에서 [승인]/[반려] → 결과 메일 자동 발송. 승인(선정)된 기업만 자료 제출 가능
4. **제출물 검토**: "제출물" 탭에서 파일 확인 → [검토] → 승인/반려/**재제출 요청**+피드백 → 피드백 메일 자동 발송
5. 제출 마감 3일/1일 전, 미제출 기업에 자동 리마인더 (CRON 설정 필요)

- 신청 기간·제출 마감은 서버에서 강제됩니다 (기간 외 신청/제출 불가)
- HWP·DOCX 첨부는 자동으로 PDF 변환되어 브라우저 미리보기 제공 (CloudConvert 설정 필요, 7장)

### 3-8. 메일 (`/admin/emails`)
- **발송 로그**: 모든 메일의 성공/실패 이력. 실패 건은 [재발송] 가능
- **새 메일**: 입주기업 다중 선택 + 제목/본문 직접 작성 발송 (`{{tenant_name}}` 변수 사용 가능)
- **템플릿 관리**: 자동 발송 메일 8종(계정 안내/청구서/납부 리마인더/프로그램 공지/선정 결과/제출 요청/검토 결과/문의 알림)의 제목·본문 수정. 사용 가능한 변수가 편집 화면에 표시됩니다

## 4. 입주기업 포털 안내 (기업 배포용)

> 아래 내용을 계정 발급 시 기업에 안내하세요.

1. **로그인**: https://www.puholdings.co.kr/portal — 안내받은 이메일·임시 비밀번호로 로그인 → 새 비밀번호 설정(8자 이상)
2. **대시보드**: 최근 청구서·미납 여부, 진행 중 프로그램 신청·제출 마감이 표시됩니다
3. **청구서**: 월별 청구 내역(임대료/관리비/전기료 항목별) 확인, **청구서 PDF 다운로드**, 인쇄
4. **프로그램**: 모집 중 공고 확인 → [신청하기] → 선정되면 같은 화면에서 파일 제출(여러 개 가능, 파일당 20MB). 재제출 요청을 받으면 다시 제출할 수 있습니다
5. **설정**: 비밀번호 변경
6. 계정 분실·오류 문의: 창업보육센터 운영 사무국

## 7. 환경변수 (기능 활성화 스위치)

Vercel 프로젝트 설정(Settings > Environment Variables)에서 관리합니다. **미설정이어도 사이트는 정상 동작**하며 해당 기능만 비활성화됩니다.

| 변수 | 용도 | 현재 | 미설정 시 |
|---|---|---|---|
| `DATABASE_URL` 등 Postgres 계열 | DB 연결 | ✅ 설정됨 | — |
| `BLOB_READ_WRITE_TOKEN` | 파일 저장 | ✅ 설정됨 | — |
| `RESEND_API_KEY` | 메일 발송 | ❌ | 모든 메일이 스킵되고 실패 로그만 기록 |
| `MAIL_FROM` | 발신 주소 (도메인 DNS 인증 필요) | ❌ | 〃 |
| `ADMIN_NOTIFY_EMAIL` | 홈페이지 문의 알림 수신 | ❌ | 문의 알림 메일 없음 (접수는 정상) |
| `CRON_SECRET` | 자동 리마인더 인증 | ❌ | 납부/제출 리마인더·연체 자동 전환 없음 |
| `CLOUDCONVERT_API_KEY` / `CLOUDCONVERT_WEBHOOK_SECRET` | HWP→PDF 변환 | ❌ | 첨부 미리보기 없음 (다운로드는 정상) |
| `APP_URL` | 변환 웹훅·메일 링크 기준 주소 | ❌ | `https://www.puholdings.co.kr` 권장 |
| `BILLING_BANK_INFO` | 청구서 PDF 납부계좌 문구 | ❌ | 기본값: "예금주: ㈜포항연합기술지주 / 하나은행 910-910009-44304" |
| `JWT_SECRET` | 로그인 서명키 (선택) | ❌ | DB 비밀번호에서 자동 파생 — **설정 불필요** |

**메일을 켜려면**: Resend(resend.com) 가입 → 도메인 인증(DNS) → API 키 발급 → `RESEND_API_KEY`, `MAIL_FROM`(예: `포항연합기술지주 <noreply@puholdings.co.kr>`), `ADMIN_NOTIFY_EMAIL` 추가 → 재배포.

## 8. 기술 운영 정보 (개발 담당자용)

| 항목 | 내용 |
|---|---|
| 저장소 | github.com/keilvher1/puholdings (main 브랜치) |
| 호스팅 | Vercel — 프로젝트 `puholdings` / `vercel --prod`로 배포 |
| DB | Neon Postgres (Vercel 연동) — raw SQL, `lib/db.ts`의 `getDb()` |
| 파일 | Vercel Blob (private) + `/api/file` 프록시 |
| 마이그레이션 | `scripts/migrations/*.sql` → `DATABASE_URL=... node scripts/run-migration.mjs <파일>` (전부 멱등, 적용 완료) |
| 테스트 | `pnpm test` — 관리비 계산엔진 실측값 검증 9건 |
| 개발 컨벤션 | 저장소 루트 `CLAUDE.md` 참고 |
| 청구서 PDF | `@react-pdf/renderer` + Pretendard 폰트(`public/fonts/`) |
| 관리자 비밀번호 재설정 | DB `admins.pass_hash`를 bcrypt 해시로 갱신 (개발 담당자 작업) |

### 보안 메모
- `/api/admin/setup`: 관리자가 이미 존재하면 로그인 없이 계정 생성 불가 (2026-07-21 적용)
- 포털은 세션의 기업 ID로만 데이터 접근 (타 기업 청구서/파일 접근 시 404)
- 첨부 경로 이동(`..`) 차단, 청구서 PDF는 관리자/해당 기업만 다운로드 가능
- 로그인 토큰 서명키는 `JWT_SECRET` 미설정 시 DB 비밀에서 파생 (소스코드에 비밀 없음)
